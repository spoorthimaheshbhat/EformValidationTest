# Test-A-Form: Business Requirements & Automation Project

## Overview
This project demonstrates end-to-end analysis, design, and testing of a data capture system using a Google Form. It combines Business Analysis and QA practices to ensure accurate data validation and seamless user interaction.

---

## Objective
To design and validate a structured e-form system ensuring:
- Accurate data capture
- Proper validation handling
- Smooth user experience
- Reliable backend storage

---

## My Role
Business Analyst + QA Engineer

- Created Business Requirements Document (BRD)
- Defined functional requirements and validation rules
- Designed end-to-end user flow
- Performed manual and automated testing

---

## Project Components

### 1. Business Requirements Document
- 7+ functional requirements (FR-01 to FR-07)
- Scope, assumptions, dependencies
- UI and data validation mapping

### 2. Test Scenarios
- Positive and Negative test cases
- Optional/Boundary test cases
- Field validation coverage
- Stored in Excel format

### 3. Automation Framework
- Selenium WebDriver
- Java
- TestNG
- Maven

---

## System Workflow

User → Open Form → Enter Data → Validation → Submit → Store in Google Sheets → Confirmation Message

---

## Key Features Tested
- Form accessibility via public URL
- Field validations (email, numeric range, required fields)
- Multi-section navigation
- Successful form submission
- Confirmation message validation
- Backend data storage verification

---

## Outcome
- Achieved full validation coverage for all required fields
- Ensured reliable data capture in Google Sheets
- Built reusable automation scripts for UI validation
